Visual Direction & Design System Brief: SocialLab Presentation WebsiteThis document outlines the visual direction, design system tokens, and front-end motion architecture required to build a high-end, slide-based interactive presentation website. This design language is strictly derived from the provided source images, including Screenshot 2026-07-06 110412.png, Screenshot 2026-07-06 110433.png, and Screenshot 2026-07-06 110447.png.1. Brand Feeling and Visual PersonalityThe website must balance a scholarly, research-driven data lab aesthetic with a humanitarian, human-centric approach to AI. It steers completely clear of cold, hyper-corporate SaaS layouts. Instead, it feels editorial, academic, and deeply intentional.Immersive Slide Presentation FeelCinematic & Deliberate: Each slide functions as a curated canvas. Information is grouped loosely but structurally, utilizing wide spatial padding to feel like an interactive exhibition.Complex yet Legible: Advanced tech and mathematical motifs coexist with soft, organic background tones, ensuring the brand feels approachable rather than intimidating.Warm Tech: By rejecting sterile dark-mode neon and stark whites in favor of warm creams and rich earth-toned teals, the presentation creates an inviting environment for storytelling.2. Typography DirectionThe typography system uses a clean, geometric, and technical sans-serif font family. It communicates high data precision through flat-sided profiles, condensed proportions, and generous letter spacing on titles.Typography SpecificationsRecommended Google Fonts:Headings: Rajdhani or Share Tech (Captured from the technical, flat-sided, geometric profiles seen across all headlines).Body Text: Barlow or Inter (Provides clean, highly legible grotesque sans-serif scaling for data-dense passages).Heading Style: Condensed, semi-geometric tracking. Major titles feature a hybrid application where alternating words switch between Lab Teal and Terracotta Coral.Letter Spacing:Headings: tracking-wide ($1.5px$ to $2px$) to elevate the tech-lab tone.Body: tracking-normal for structural clarity.Font Weights: Title elements utilize Medium (500) to Bold (700). Body text remains strictly at Regular (400) to prevent visual clutter against light backdrops.Casing Style: Sentence case or Title Case for primary presentation slides; explicit uppercase transforms are reserved for micro-section anchors (e.g., "OPERATIONS" as seen in Screenshot 2026-07-06 110412.png).4. Slide/Section Structure: Every section must enforce full-bleed viewport heights (w-full h-screen). UI Component StyleComponents are flat, clean, and map directly to an interactive presentation deck environment.Navbar: Fixed, high-position header using zero backdrop shadows. It sits on a clear or transparent background, relying on simple padding (py-6).Active Navigation Item: Text color transitions to Terracotta Coral with a thin, sharp horizontal underscore indicator (border-b-2) positioned directly below the active route.Buttons:Contoured pill capsules (rounded-full).Style A (Outline): Thin, neutral border with an integrated search or arrow glyph.Style B (Solid): Soft warm base fills (e.g., Muted Amber or Terracotta Coral) with clean technical text overlays. No loud shadows.Cards (Recent Innovations & Blogs): Pure flat white background blocks with low-radius corners (rounded-lg). Borders are thin and light (border-gray-100), ensuring the cards look like organized documents rather than floating layers.Progress Indicator: A floating, minimalist navigation rail pinned to the right screen edge (Screenshot 2026-07-06 110412.png). It displays discrete, vertical color blocks representing the active deck section (Teal, Coral, and Peach bars).6. Visual Composition PrinciplesAll custom SVGs (Screenshot 2026-07-06 110433.png, Screenshot 2026-07-06 110511.png) must be constructed programmatically using these elements:Thin Vector Paths: Fine strokes (stroke-[1.5px]) utilizing Lab Teal or Terracotta Coral.Perfect Circular Arcs: Concentric, unclosed circular lines that frame inner data concepts.Vertex Node Dots: Small, filled circle coordinates (r-1.5 or r-2) anchoring line intersections.Abstract Motifs: Human profiles integrated with circuit traces, neural net webs, mathematical fractions ($V = 1/3$), and floating square brackets.Underlying Backdrop Blobs: Large, organic, soft-filled circular geometries using highly transparent variations of peach and coral, layered beneath the sharp vector paths.7. Animation and Interaction DirectionMotion must remain premium, sophisticated, and calm. Avoid aggressive elastic easing, bouncing, or flashy gaming transitions.Scroll Snapping Behavior: Enforce strict CSS vertical scroll snapping (snap-y snap-mandatory) across the main container workspace to lock slides into view cleanly.Section Reveal Transitions: Content within a slide enters view using an upward directional fade:JavaScript{ initial: { opacity: 0, y: 30 }, animate: { opacity: 1, y: 0 }, transition: { duration: 0.8, ease: [0.25, 1, 0.5, 1] } }
SVG Stroke Drawing: When a slide activates, its vector artwork lines animate into view via a stroke-drawing effect (pathLength: [0, 1]) across a $1.5\text{-second}$ timeline.Ambient Floating Elements: Background decorative blobs use a gentle loop animation to feel organic:JavaScriptanimate={{ y: [0, -8, 0] }} transition={{ repeat: Infinity, duration: 6, ease: "easeInOut" }}
Stat Count-up Numbers: Automatically trigger an interpolation counter for metrics (60+, 11) when the viewport registers entry into a dark teal slide section.8. Layout Component Architecture (SlideContainer.jsx)JavaScriptimport { motion } from 'framer-motion';

export const SlideContainer = ({ children, bgColor = "bg-brand-cream" }) => {
  return (
    <section className={`w-full h-screen snap-start flex items-center justify-center relative overflow-hidden ${bgColor}`}>
      <motion.div 
        initial={{ opacity: 0, y: 40 }}
        whileInView={{ opacity: 1, y: 0 }}
        viewport={{ once: true, margin: "-100px" }}
        transition={{ duration: 0.8, ease: [0.25, 1, 0.5, 1] }}
        className="max-w-7xl w-full mx-auto px-8 grid grid-cols-12 gap-8"
      >
        {children}
      </motion.div>
    </section>
  );
};

Do and Do Not ListDODO prioritize full-screen vertical scroll snapping (snap-y) to maintain the feel of a presentation slide deck.DO use two-tone heading colors (Lab Teal mixed with Terracotta Coral) inside major titles.DO write clean, native inline SVG paths for icons, animating lines with pathLength changes.DO stick to Warm Cream and flat Pure Laboratory White backgrounds to ensure optimal content legibility.DO use wide letter-spacing (tracking-wide) on technical headings to reinforce the clean, data-lab atmosphere.DO NOTDO NOT use default modern SaaS elements like heavy glassmorphism, glowing dark modes, or vibrant neon gradients.DO NOT introduce loud shadows, floating cards with high elevations, or bright gradient buttons.DO NOT use generic icon libraries (such as FontAwesome or Lucide) for central graphics. Stick to custom geometric vector shapes.DO NOT create long scrolling layouts. Break content down into structured, standalone slide viewports.
