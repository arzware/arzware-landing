1. Brand Feeling and Visual Personality

The reference website feels like human-centered AI, research, social impact, and technical innovation.

The presentation website should feel:

Intelligent but warm → not cold enterprise AI.
Scientific but human → diagrams, nodes, data arcs, human/brain motifs.
Academic + innovation lab → like an AI research studio presenting frameworks and social-good projects.
Calm and deliberate → lots of space, soft backgrounds, slow motion, structured layouts.
Retro-futuristic → geometric typography, muted colors, technical illustrations, thin lines.
Presentation-native → every section should feel like a curated slide, not a long scrolling landing page.

The visual language is not “startup SaaS.” It is closer to:

AI lab + social innovation studio + interactive research presentation.

3. Typography Direction

The typography is one of the strongest brand signals.

Visual Style

The screenshots use a type style that feels:

Condensed
Rounded-geometric
Monospaced-adjacent
Slightly futuristic
Technical but playful
Wide letter spacing
Light to medium font weight
Recommended Fonts

Best Google Font options:

Rajdhani
Closest practical match.
Good for headings, nav, buttons, labels.
Has the condensed tech look.
Share Tech Mono
Good for smaller labels and technical UI text.
Use carefully, not for everything.
Orbitron
Too futuristic if overused.
Could work only for tiny labels, but I would avoid it for main text.
Oxanium
Strong alternative.
Feels technical and rounded.

Recommended pairing:

Headings / Nav / Buttons: Rajdhani
Body / Paragraphs: Rajdhani or Inter
Technical labels: Share Tech Mono

Better practical setup:

font-family: "Rajdhani", system-ui, sans-serif;
Heading Style

Use:

Large size
Light/medium weight
Wide tracking
Mixed color words
Sentence case or title case
Avoid bold heavy headings

Example style:

.heading-xl {
  font-family: "Rajdhani", sans-serif;
  font-weight: 500;
  letter-spacing: 0.12em;
  line-height: 1.05;
}
Body Style

Use:

Small to medium size
Teal-grey text
Generous line height
Slight tracking
Narrow content width

Example:

.body {
  font-family: "Rajdhani", sans-serif;
  font-weight: 400;
  letter-spacing: 0.04em;
  line-height: 1.75;
}
Casing

Use mostly:

Title Case for section titles
ALL CAPS for labels like OPERATIONS, SERVICES, OVERVIEW
Short technical labels for icons

Avoid generic SaaS copy styling like massive bold black headlines.

4. Layout Rules
Overall Structure

The site should behave as a slide-based scroll experience.

Each section should be:

min-height: 100vh
scroll-snap-align: start
large visual composition
limited text
strong spatial rhythm

Recommended structure:

Slide 1: Hero / Brand intro
Slide 2: Vision / Concept
Slide 3: Services / Capabilities
Slide 4: Roadmap / Process
Slide 5: Projects / Innovations
Slide 6: Industries
Slide 7: Clients / Partners
Slide 8: Closing / Contact
Slide Layout

Use a consistent max-width:

max-width: 1280px to 1440px
horizontal padding: 5vw to 7vw
vertical padding: 96px to 140px

The reference uses a lot of wide horizontal breathing room. Do not crowd sections.

Grid Behavior

Use:

12-column desktop grid
Large empty zones
Content often centered or slightly left aligned
Visuals often oversized and partially cropped
Cards in 2-column or 3-column groups
Icon sections in 4 to 6 equal columns

Example:

Hero:
left 40% text
right 60% abstract SVG system

Roadmap:
4 equal columns
each with icon → line → number → title → paragraph

Services:
6 icon blocks in one horizontal row

Projects:
left feature card
right stacked screenshots/cards
Alignment Style

The design avoids strict corporate symmetry. It uses:

Centered headings
Asymmetric hero visuals
Wide horizontal icon rows
Footer with structured columns
Large visual elements crossing section boundaries
Hero Layout

Hero should have:

Warm sand background
Logo/nav fixed at top
Large typography on left
Abstract AI/data illustration on right
CTA pills below title
Subtle decorative nodes and arcs
Stats Section

Stats should be:

Full-width dark teal
Large numbers
Minimal labels
Low-contrast oversized typography
Strong horizontal distribution

Use large numbers but avoid making it look like generic KPI blocks.

Footer Style

Footer should be:

Deep teal background
Multiple text columns left
Large logo or wordmark right
Small circular social icons
Thin bottom copyright strip

Stock icon packs

3D icons
Neon gradients
Realistic AI robots

The icons should look drawn from the brand’s internal visual grammar.

7. Animation and Interaction Direction

Motion should feel calm, intelligent, and presentation-like.

Scroll Behavior

Use:

scroll-snap-type: y mandatory;
scroll-behavior: smooth;

Each slide:

scroll-snap-align: start;
min-height: 100vh;

For a more controlled premium feel, use Framer Motion with section index tracking.

Section Transitions

Each slide should transition with:

Soft opacity fade
Slight vertical movement
SVG elements drawing in
Background color crossfade
No harsh page jumps

Example:

section enters:
opacity 0 → 1
y 40 → 0
duration 0.8s
ease: cubic-bezier(0.22, 1, 0.36, 1)
Reveal Animations

Use staggered reveals:

heading first
subtitle second
visual third
buttons last

For icon rows:

icons reveal one by one with 80ms–120ms stagger
SVG Stroke Drawing

Use Framer Motion path animations:

<motion.path
  initial={{ pathLength: 0, opacity: 0 }}
  whileInView={{ pathLength: 1, opacity: 1 }}
  transition={{ duration: 1.4, ease: "easeInOut" }}
/>

This is very important for matching the technical diagram feeling.

Floating Decorative Elements

Use subtle motion only:

node dots drift 4–8px
arcs rotate 1–3 degrees
small labels fade in/out softly

Duration:

6s–12s
easeInOut
repeat: Infinity
repeatType: reverse

Avoid fast movement.

Hover States

Buttons:

translateY(-1px)
slight opacity change

Cards:

image opacity increases
border color becomes coral/teal
small visual shift

Nav:

underline slides in
text color coral

Icons:

arc rotates slightly
coral node pulses softly
Stat Count-Up

Stats should animate when visible:

0 → final number
duration: 1.2s
easeOut

Keep it subtle. No casino-style number rolling.

Parallax / Layer Motion

Use light parallax on hero only:

background blob moves slow
teal arc moves slightly faster
node network moves independent

Do not parallax text heavily.

Slide Component
import { motion } from "framer-motion";

type SlideProps = {
  id: string;
  className?: string;
  children: React.ReactNode;
};

export function Slide({ id, className = "", children }: SlideProps) {
  return (
    <section
      id={id}
      className={`presentation-slide px-[6vw] py-28 ${className}`}
    >
      <motion.div
        initial={{ opacity: 0, y: 36 }}
        whileInView={{ opacity: 1, y: 0 }}
        viewport={{ once: false, amount: 0.45 }}
        transition={{ duration: 0.75, ease: [0.22, 1, 0.36, 1] }}
        className="mx-auto max-w-[1440px]"
      >
        {children}
      </motion.div>
    </section>
  );
}

10. Do and Do Not List
Do
Use warm sand, teal, coral, peach, and off-white.
Use condensed geometric typography.
Use large letter spacing.
Build full-screen scroll-snapping sections.
Use custom SVG illustrations.
Animate SVG paths drawing in.
Use subtle staggered reveals.
Keep layouts spacious.
Use pill buttons.
Use thin active nav underlines.
Use right-side slide progress.
Mix technical diagrams with human-centered visuals.
Use muted, calm, premium motion.
Let some illustrations partially crop or overlap section boundaries.
Keep section copy minimal and presentation-like.
Do Not
Do not make it look like a generic SaaS landing page.
Do not use neon colors.
Do not use blue/purple AI gradients.
Do not use heavy glassmorphism.
Do not use loud shadows.
Do not use 3D AI robots.
Do not use generic stock icons.
Do not use heavy bold fonts.
Do not overcrowd slides with content.
Do not copy the screenshots pixel-for-pixel.
Do not use random colors outside the reference palette.
Do not make animations bouncy or fast.
Do not turn it into a PowerPoint-style deck.
Do not make every section white.
Do not treat it as a normal landing page.
